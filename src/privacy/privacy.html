Privacy Policy

Send to Supernote · Last updated 2026-06-13

Send to Supernote captures the current web page, converts it to a PDF or EPUB entirely on your device, and uploads it to your Supernote destination — the public Supernote Cloud or your own self-hosted Private Cloud. A PDF you are already viewing is sent as-is. Nothing passes through any server we run.

What we store, and where

  • A session token only. When you connect, the access token is stored in chrome.storage.local on this device only. Your password is never stored — for Private Cloud it is used in memory only to compute the login hash and then discarded; for Supernote Cloud you sign in on Supernote's own page and the extension only reads the resulting session cookie.
  • No chrome.storage.sync — nothing sensitive is synced across machines.
  • Local settings (format, target, destination folder, filename-confirm, and the Add source & time toggle) and a short local send history, stored locally for your convenience. None of it leaves your device except as part of a send you initiate.

Source URL & capture time (opt-in, off by default)

  • The popup has an Add source & time toggle that is off by default. When enabled, each send stamps the page's original URL and the capture time into the file (a small visible header plus file metadata). It is opt-in because it embeds your browsing source; the preference is stored locally only, and the URL goes only to the Supernote destination you choose — never to any third party.
  • The full URL is embedded verbatim, including any query parameters — which can sometimes carry session tokens or one-time links. Leave the toggle off (the default) for such pages, or strip sensitive query parameters before sending.

Where your data goes

  • Page content is converted locally and uploaded only to the destination you choose:
  • Supernote Cloud — viewer.supernote.com / cloud.supernote.com and Ratta's own storage (Amazon S3) via a pre-signed URL Supernote's API issues; or
  • Your Private Cloud server — the exact base URL you configured; nothing leaves your network beyond your own server.
  • No third-party providers (no Dropbox/Drive/OneDrive), no third-party sign-in (no OAuth), and no server operated by this project.
  • No telemetry, no analytics, no remote logging.

Your control

  • Disconnect at any time to remove the stored token from this device.
  • Uninstalling the extension removes all locally stored data.
